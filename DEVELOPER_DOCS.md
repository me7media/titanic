# Технічна Документація Розробника (Developer Docs)

Проект **Титанік 3D** — це процедурно згенерована симуляція-пісочниця на базі Three.js, що працює повністю у браузері без необхідності компіляції серверного бекенду. Гра використовує ванільний JavaScript, `type="module"` та вбудовані патерни проектування для управління 3D-сценою.

## Технічний Стек
- **Рушій**: [Three.js r128](https://threejs.org/) (завантажується через CDN).
- **Стилі**: Чистий CSS (Grid / Flexbox для UI-оверлею).
- **Пакетний менеджер**: Проект поставляється "як є" (може виконуватись через `npx serve`), без бандлерів (Webpack/Vite).

## Архітектура (Entity-Component-System Pattern)
Гра уникає глибоких ієрархій ООП та використовує глобальний State-об'єкт із зовнішніми "резолверами" (модулями), які виконують фізику та рендер:
- `state.js`: Глобальний об'єкт `game`. Зберігає фази (`sailing`, `sinking`), координати гравців, енергію, стан айсбергів та кімнат.
- `main.js`: Вхідна точка (`entrypoint`). Відповідає за `requestAnimationFrame` цикл, слухачі клавіатури, налаштування камери та оновлення UI (health bars, mood).
- `environment.js`: Процедурна генерація хвиль і айсбергів. Використовує маніпуляцію вертексами `PlaneGeometry` на базі синусоїд для імітації штормового океану. Спавнить айсберги із рандомною DodecahedronGeometry.
- `ship.js`: Найтовстіший модуль. Будує корабель процедурно:
  - `getHullZ()` — математична функція профілю корпусу (товщина геометрії залежить від X-позиції).
  - Складає корабель з двох масивних груп (`bowGroup` та `sternGroup`), що дозволяє розірвати корабель на дві незалежні фізичні частини в моменти часу > 60 секунд після зіткнення. 
- `characters.js`: Процедурний "ріггінг" (rigging) персонажів. Голови, тулуб і кінцівки зберігаються в ієрархії `THREE.Group`. Функція `updateCharacterPose` застосовує кути Ейлера (`Euler angles`) до цих підгруп для симуляції анімацій (сидіння, харчування, лежання).

## Фізика та Навігація
Замість складної системи `Raycaster` (яка дорога для CPU в JS), гра використовує 2D Bounding Boxes. Коли персонаж рухається або стрибає, його координати `(X, Z)` затискаються (clamp) між фізичними параметрами вибраної палуби:
```js
mesh.position.x = Math.max(bounds.minX, Math.min(mesh.position.x, bounds.maxX));
mesh.position.z = Math.max(-bounds.zMax, Math.min(mesh.position.z, bounds.zMax));
```

## Додавання нових фіч: Шпаргалка
1. **Нова кімната**: Додайте її назву у `ROOM_ORDER` та її Y-зміщення у `ROOM_Y_POSITIONS` в `state.js`.
2. **Нова анімація гравця**: Додайте цільові кути в `updateCharacterPose` (`characters.js`) за назвою (напр., `pose === 'dance'`).
3. **Зміна тривалості "Життя"**: `game.time` рахується в кадрах * 0.016, що приблизно дорівнює секундам. Усі події сценарно зашиті через `game.time > 180` (Айсберги) або `timeSinceHit > 60` (Розлом корпусу).
