# Технічна Документація Розробника (Developer Docs)

Проект **Титанік 3D** — це процедурно згенерована симуляція-пісочниця на базі Three.js, що працює повністю у браузері без необхідності компіляції серверного бекенду. Гра використовує ванільний JavaScript, `type="module"` та вбудовані патерни проектування для управління 3D-сценою.

## Технічний Стек
- **Рушій**: [Three.js r128](https://threejs.org/) (завантажується через CDN).
- **Стилі**: Чистий CSS (Grid / Flexbox для UI-оверлею) з використанням Glassmorphism.
- **Введення**: [NippleJS](https://yoannmoinet.github.io/nipplejs/) для мобільного джойстика.
- **Пакетний менеджер**: Проект поставляється "як є" (запуск через `npx serve`), без бандлерів.

## Архітектура (Entity-Component-System Pattern)
Гра уникає глибоких ієрархій ООП та використовує глобальний State-об'єкт із зовнішніми "резолверами" (модулями), які виконують фізику та рендер:
- `state.js`: Глобальний об'єкт `game`. Зберігає фази (`sailing`, `sinking`), координати гравців, енергію, стан айсбергів та кімнат.
- `main.js`: Вхідна точка (`entrypoint`). Відповідає за `requestAnimationFrame` цикл, налаштування камери та оновлення UI (health bars, mood).
- `input.js`: **Уніфікований шар введення**. Абстрагує клавіатуру, мишу та тач-панелі мобільних пристроїв у єдиний стан `game.keys`.
- `environment.js`: Процедурна генерація хвиль і айсбергів. Використовує маніпуляцію вертексами `PlaneGeometry` на базі синусоїд.
- `ship.js`: Процедурне будівництво корабля (**220x32x18**). Містить допоміжні функції `getSheer(x)` (вигин палуби) та `getHullZ(x, y, baseZ)` (динамічна ширина борту) для точного кріплення такелажу та деталей.
- `characters.js`: Процедурний "ріггінг" персонажів та анімації через кути Ейлера.

## Система Керування та Камера
Гра підтримує два типи керування:
1. **Корабель**: Орбітальна камера з "Peek"-ефектом (залежить від `mouseX/Y`).
2. **Персонаж**: Камера від третьої особи з плавним слідуванням та кінематографічним режимом "Fly" (shot від першої особи Роуз при натисканні 'L').

## Мобільна Адаптація
- **Джойстик**: `nipplejs` мапує рух на клавіші `w, a, s, d`.
- **Жести**: Реалізовано "Pinch-to-zoom" (зміна `game.camDist`) та "Swipe-to-look" (зміна `game.mouseX/Y`).
- **Контекст**: Кнопки дій (`Button F, X, Z, L`) з'являються динамічно залежно від поточної кімнати.

## Тестування та Валідація
Для стабільності розробки використовується `test.html`. Він виконує понад 10 автоматизованих тестів:
- Цілісність модулів та референсів.
- Фізика корабля та гальмування.
- Механіки виживання (Hunger, Warmth, Mood).
- Перемикання режимів та кімнат.
- Логіка близькості персонажів (сцена "Я лечу").
- **Система Станів (Win/Loss)**: Керується через `gameStatus` ('playing', 'won', 'lost').
- **Restart Timer**: При досягненні фіналу (перемога чи поразка) запускається 15-секундний відлік до `location.reload()`.
- **Зона Шлюпки**: Визначається через `dist < 12` від об'єкта `rescueBoat`. Всередині цієї зони стат-деградація відключається.
- **Діагностика**: Команда `window.runDiagnostics()` запускає повний цикл тестів прямо в рантаймі (повідомлення, спавн айсбергів, занурення носом).
- **Сценарне занурення**: Підтримка кута **-1.57** (90°) для корми з уповільненою інтерполяцією для ефекту масивності.

## Додавання нових фіч: Шпаргалка
1. **Нова кімната**: Додайте назву у `ROOM_ORDER` та Y-зміщення у `ROOM_Y_POSITIONS` в `state.js`.
2. **Новий мобільний контекст**: Оновіть `updateMobileContext()` в `input.js` для відображення кнопок.
3. **Нова анімація**: Оновіть `updateCharacterPose()` в `characters.js`.
4. **Валідація**: Після змін обов'язково запустіть `test.html` для перевірки регресій.
